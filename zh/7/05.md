`changes`有一堆辅助函数，我们可以使用它们来修改游戏状态的特定部分。

例如，如果你想将1号玩家的Overlord起始防御值从20更改为30，你将执行以下操作：

```
changes.changePlayerDefense(Player.Player1, 30);
```

想知道那个`Player.Player1`是什么东西？

它是来自`ZBGameMode.sol`的自定义`enum`数据类型：

```
enum Player {
    Player1,
    Player2
}
```

不熟悉的话可以了解下：在Solidity中，`enum`是一种创建用户定义数据类型的方法，可以轻松地和`uint`相互转换。

`enum`可以按如下方式转换为`uint`：

```
uint(Player.Player1); // == 0
uint(Player.Player2); // == 1
```

`uint`也可以转换为`Player`:

```
Player(0); // == Player.Player1
Player(1); // == Player.Player2
```

我们为什么要这样做？ 主要是为了代码可读，以及更清晰地了解实际发生的情况。

例如，这段代码：

```
changes.changePlayerDefense(Player.Player1, 30);
```

……比起下面这个可读性要高多了：

```
changes.changePlayerDefense(0, 30);
// What is 0???
```

在第一个示例中，我们很容易看到我们正在修改1号玩家的数据，而在第二个示例中，读取代码的人需要查找函数定义才能看到`0`所指的内容。

## 更改“僵尸战场”玩家数据

除了上面的`changes.changePlayerDefense（）`之外，我们还可以在`changes`对象上调用许多其他方法来调整游戏中的各种数据：

* `changePlayerCurrentGooVials`  —— 玩家当前可用于此回合的小瓶数量。 比赛开始时默认值从1开始，每一回合增加1。
* `changePlayerCurrentGoo`  —— 当前回合中用装满goo的小瓶数量。 默认值与每个回合开始时的goo小瓶数量相同。
* `changePlayerMaxGooVials`  —— 玩家在游戏中可以拥有的goo小瓶的最大数量。 默认值为10。
* `changePlayerInitialCardsInHandCount`  —— 玩家在比赛开始时的牌数。 默认值为3。
* `changePlayerMaxCardsInPlay`  —— 玩家可以投入到场上的最大僵尸数。 默认值为6。
* `changePlayerMaxCardsInHand`  —— 玩家一次可以握在手中的最大牌数。 默认值为10

这些函数都遵循与`changePlayerDefense`相同的格式，并接受两个参数：`Player` enum,和`uint8`。
