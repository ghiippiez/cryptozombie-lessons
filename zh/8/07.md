很好！现在我们已经申明了2个新的空牌组，每个玩家一个。

下一步，逐张浏览每个玩家的牌，生成一个随机数相当于抛硬币，然后使用它来确定哪个玩家获得此卡牌。

在Solidity中多次（伪）抛硬币的方法如下：

```
uint nonce = 0;
for (uint i = 0; i < 10; i++) {
    // The `% 2` at the end means "divide by 2 and take the remainder", so `rand` will end up
    // equaling either `0` or `1`
    uint rand = uint(keccak256(abi.encodePacked(now, nonce))) % 2;

    if (rand == 0) {
      // do something
    } else {
      // do something else
    }

    // We need an incrementing nonce, to make sure `rand` is different each time we run the loop
    // Otherwise the result of the random would be the same in every interation
    nonce++;
}

```

 如果你需要复习随机数，可以查看CryptoZombies第一季的<a href="https://cryptozombies.io/en/lesson/4/chapter/4" target=_blank>第4课第4章</a>。

> 注意/提醒：这种生成伪随机数的方法不应该用于赌博游戏，因为区块生产者可以通过选择不发布事务来钻空子，除非它让他们赢得一大笔钱。 但对于像我们这样简单的洗牌来说，它已经足够好了。

## 实战演习

1. 首先，删除我们在**Singleton**中实现的内部`for`循环的全部内容，这样就可以用我们自己的逻辑来替换它了。

2. 我们的第一步是生成一个名为`rand`的随机`uint`，其值为0或1，就像我们上面所做的那样。 但是，我们可以使用`player1CardCount + player2CardCount`来代替上面的例子中的`nonce`，因为我们知道每次运行`for`循环时这两个值中的一个就会递增。

3. 现在生成了一个随机数，我们可以将该卡牌添加到其中一个玩家的牌组中。

    a. 创建一个`if` / `else`语句，检查`rand`是否等于`0`。
    
    b. 如果是，我们就将该卡牌添加到1号玩家的牌组中。 所以在if语句中，将`player1Cards[player1CardCount]`设置为等于`gameState.playerStates[i].cardsInDeck[j]`。
    
    c. 同样在`if`语句中，使用`player1CardCount++`增加`player1CardCount`。

    d. 在`else`语句中，执行相反的操作：将卡牌添加到**2号玩家**的牌组中，并增加`player2CardCount`。
